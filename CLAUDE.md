## Назначение репозитория
**Экран очереди в кофейне** — большой дисплей, на котором клиенты видят актуальные статусы заказов.
Только **отображение** (read-only): своих мутаций нет, статусы меняет бариста через backend.

## Стек
- **React + Vite + TypeScript** (SPA).
- В инкременте 1 — локально, без деплоя.

## Как работает (границы инкремента 1)
1. При загрузке — **снапшот** очереди по REST: `GET /orders` (backend).
2. Дальше — **подписка на SSE**: `GET /orders/stream` — сервер шлёт актуальные снапшоты, экран перерисовывается live. ADR 0001.
3. **Типы `Order` — генерируются из OpenAPI backend** (зафиксированная версия спеки), руками не пишем. ADR 0008.

## Как связан с соседями
- Полностью зависит от контракта `brewline-backend` (OpenAPI = источник истины).
- Адрес API — через env (`VITE_API_URL`); хардкодить URL нельзя.

## Локальный запуск (наполнится по мере кода)
- `npm run dev` (Vite).
- Требует запущенный backend + Mongo (из `brewline-infra`).

## Безопасность (репо публичный)
- Не коммитить `.env`. В код — только `VITE_*` плейсхолдеры и `.env.example`.
  Помни: всё, что попадает в `VITE_*`, уходит в браузерный бандл — секретов туда не класть в принципе.

## Конвенции
- TypeScript strict; типы контракта — только из сгенерированных из OpenAPI, не дублировать вручную.
