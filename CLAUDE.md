## Назначение репозитория
**Экран очереди в кофейне** — большой дисплей, на котором клиенты видят актуальные статусы заказов.
Только **отображение** (read-only): своих мутаций нет, статусы меняет бариста через backend.

## Стек
- **React + Vite + TypeScript** (SPA).
- В инкременте 1 — локально, без деплоя.

## Как работает (границы инкремента 1)
1. При загрузке — **снапшот** очереди по REST: `GET /orders` (backend).
2. Дальше — **подписка на SSE**: `GET /orders/stream` — сервер шлёт актуальные снапшоты, экран перерисовывается live. ADR 0001.
3. **Типы `Order` — из пакета `@brewline/api-types`** (сгенерированы из OpenAPI backend), руками не пишем. ADR 0008, 0009.

## Как связан с соседями
- Полностью зависит от контракта `brewline-backend`. Контракт для нас — OpenAPI-спека; сама она
  генерируется у backend из zod-схем (ADR 0010), но нас это не касается: мы видим только пакет типов.
- Адрес API — через env (`VITE_API_URL`); хардкодить URL нельзя.

## Контракт (типы из npm-пакета)
- Типы приходят из **`@brewline/api-types`** — пакет собирается из OpenAPI-спеки в `brewline-backend`
  и публикуется в npmjs.com (ADR 0009). Вендорной копии спеки и шага генерации здесь **нет**.
- Пакет **types-only** → живёт в `devDependencies`, в бандл не попадает.
- Импорт — только `import type { Order } from '@brewline/api-types'`.
- **Обновление контракта:** `npm outdated @brewline/api-types` покажет отставание →
  `npm i -D @brewline/api-types@<версия>`. Major-версия = несовместимое изменение API.

## Локальный запуск (наполнится по мере кода)
- `npm run dev` (Vite).
- Требует запущенный backend + Mongo (`docker compose up` в `brewline-backend`).

## Безопасность (репо публичный)
- Не коммитить `.env`. В код — только `VITE_*` плейсхолдеры и `.env.example`.
  Помни: всё, что попадает в `VITE_*`, уходит в браузерный бандл — секретов туда не класть в принципе.

## Конвенции
- TypeScript strict; типы контракта — только из `@brewline/api-types`, не дублировать вручную.
