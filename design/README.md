# Дизайн-макет: табло очереди в зале

Макет экрана `display`, сгенерирован в **Claude design** и реализован в `src/ui/`.

- **Превью:** `queue-screen.webp`
- **Исходник:** `queue-screen.dc.html` (+ `support.js`) — открывается в браузере; формат Claude design (canvas), с вариантами (переполнение, тёмная/светлая тема).
- **Проект в Claude design:** https://claude.ai/design/p/5c49f7d7-65fe-4419-a491-83332caf5c72
- **Дата:** 2026-07-15

## Промпт (суть)

Табло заказов в зале кофейни — большой дисплей, клиенты смотрят издалека. 3 колонки по статусам (Новый / Готовится / Готов), крупные номера, тёмная тёплая палитра, 16:9. Полный промпт — в истории обсуждения.

## Ключевые решения при реализации

- **Токены** макета (палитра, шрифты, радиусы) → CSS-переменные в `src/index.css`.
- Стили — **SCSS Modules**, colocation с компонентами (`ui/Order*.module.scss`).
- Шрифты Space Grotesk + Manrope через Google Fonts (`index.html`); self-hosted `@fontsource` — возможное упрочнение для экрана в зале.
- **Колонка «Готов»:** макет показывает `ready` («заберите на баре»), поэтому backend расширен — `GET /orders` отдаёт `new + preparing + ready` (`BOARD_STATUSES`), отдельно от активной работы бариста (`ACTIVE_STATUSES`). `ready` пока копится (упрощение walking skeleton).
- Пока не реализовано: светлая тема (вариант макета), анимация появления/ухода карточек, живое обновление (I13 — SSE).
